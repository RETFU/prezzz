Sylius

  • Projet initié par Paweł Jędrzejewski en 2012
  • Framework E-commerce basé Symfony
  • Bundles et composants indépendents les uns des autres

Kézako ?

  • Permet la gestion des ressources dans une applications via CRUD
  • Permet la mise en place d'une API
  • Inspirer d’un composant Spring (Resources)

Pourquoi ?

  • Eviter la duplication de code dans nos contrôleurs
  • Utiliser une solution simple et flexible sans utiliser d'admin generator

Comment créer une API REST ?

Comment ça fonctionne ?

  • Le ResourceController étend the FOSRestController
  • La sérialization est faite grâce au JMSSerializerBundle
  • Possibilité d'utiliser BazingaHateoasBundle

Expose en plus de l'état, les actions / transitions possibles autour de l'objet au travers de liens hypermédias.

Hypermedia As The Engine Of Application State permet de faire un API de niveau 3 Hypermedia Controls

  • implémentation découplée
  • plus besoin de manipuler les URLs
  • forme d'auto-documentation de l’API

Le routing

Utiliser le ApiLoader pour générer vos routes automatiquement

        
live_api_user_(index|show|create|update|delete):
    pattern: (/users/|/users/{id})
    methods: [(GET|GET|POST|PUT-PATH|DELETE)]
    defaults:
        _controller: live.controller.user:indexAction

Les routes sont construites à partir des resources configurées.

  • live: nom de l'application
  • user: nom de le ressource
