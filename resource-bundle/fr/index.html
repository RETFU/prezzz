Sylius

  • Projet initié par Paweł Jędrzejewski en 2012
  • Framework E-commerce basé Symfony
  • Ensemble de bundles et composants utilisables idénpendement les uns des autres

Kézako ?

  • Permet de gérer les ressources de son application (Web/Api)
  • Inspirer d’un composant Spring (Resources)

Quelques chiffres!

  • 57 Stars, 45 Forks
  • 57 contributeurs (@lsmith77, @Seldaek)

Pourquoi ?

  • Eviter la duplication de code dans nos contrôleurs
  • Utiliser une solution simple et flexible sans utiliser d'admin generator

Configurons nos ressources

    resources:
        live.user:
            driver: doctrine/orm
            object_manager: default
            templates: App:User
            classes:
                model: Live\Entity\User

Vous avez dis flexible ?

Utiliser un template différent

        
live_user_create:
    pattern: /new
    methods: [GET, POST]
    defaults:
        _controller: live.controller.user:createAction
        _sylius:
            template: LiveBundle:Backend/User:index.html.twig

Rédiriger l'utilisateur

        
live_user_create:
    pattern: /new
    methods: [GET, POST]
        _controller: live.controller.user:createAction
        _sylius:
            redirect: live_user_index
            # Ou
            redirect:
                route: live_user_index
                parameters: []

Configurer un formulaire différent

        
live_user_create:
    pattern: /new
    methods: [GET, POST]
    defaults:
        _controller: live.controller.user:createAction

Comment créer un bundle ?

Configuration sémantique

        
bundle_name:
    driver: doctrine/orm
    object_manager: my_custom_manager
	templates:
		user: LiveBundle:Backend/User
    validation_groups:
        user: [live]
    classes:
        my_entity:
            model: Live\UserBundle\Model
            controller: Live\UserBundle\Entity\ModelController
            repository: Live\UserBundle\Repository\ModelRepository
            form: Live\UserBundle\Form\Type\FormType
